# ISO Annex B Numerical Error Analysis - Rust+AVX2 vs Python/mosqito

Date: 2026-07-04
Workspace: D:\ISO532
Reference implementation: Python mosqito 1.2.1 golden data generated by `tools/gen_golden.py`
Rust path: `iso532` crate, current Rust+AVX2 optimized ZWTV pipeline
ISO reference source: Annex B CSV/XLSX data in `data/annexb/`

## Summary

Relative to ISO Annex B reference values, Rust+AVX2 and Python/mosqito are numerically indistinguishable at the ISO compliance scale. The observable ISO-reference delta is dominated by the ISO Annex B tolerance/reference layer and mosqito reference semantics, not by Rust, AVX2, rayon, or the recent layout optimizations.

Rust+AVX2 adds only a tiny residual error relative to Python/mosqito:

- Total loudness `N` / `N(t)`: 0 to about `1.6e-14` sone against mosqito golden.
- Specific loudness `N_specific`: worst observed about `1.3e-9` sone/Bark against mosqito golden.
- AVX2 vs Rust scalar `N(t)`: bit-identical over the measured golden set.
- AVX2 vs Rust scalar `N_specific(t)`: worst observed about `1.4e-9` sone/Bark.

These residuals are many orders of magnitude smaller than ISO Annex B compliance tolerance, which is `abs(error) <= 0.1 + 0.05 * abs(reference)`.

## Theoretical Error Scale

| Layer | Typical scale |
|---|---:|
| ISO Annex B compliance tolerance | `0.1 + 5%` of reference |
| `calc_slopes` total loudness quantization for `N <= 16` | max `0.0005` sone |
| `calc_slopes` total loudness quantization for `N > 16` | max `0.005` sone |
| Rust+AVX2 vs Python/mosqito total loudness residual | `0` to `1e-14` sone |
| Rust+AVX2 vs Python/mosqito specific loudness residual | `1e-14` to `1e-9` sone/Bark |
| f64 ULP noise around 1 to 10 sone | about `1e-16` to `1e-15` |

The practical conclusion is that Rust+AVX2 is numerically equivalent to Python/mosqito for this project. Any visible delta against ISO Annex B reference values is the reference/compliance layer delta already present in mosqito, not an error introduced by the Rust+AVX2 optimization work.
