4. 誤差與精度如何比較

測試誤差不是「聲音品質差多少」，而是「本系統輸出與參考輸出差多少」。比較時要先指定參考基準：Mosqito golden 用於日常回歸與相容性；ISO Annex B 用於合規檢查。

公式

絕對誤差：abs(ours - reference)

相對誤差：abs(ours - reference) / abs(reference)

`zwtv` 是時間序列，因此通常取所有時間點、所有 Bark 點中的最大值。

判讀順序

1. 先看總響度 `N` 或 `N(t)`。

2. 再看 `N_specific` 的最大絕對誤差。

3. 同時檢查相對誤差，避免 near-zero 區域被放大誤讀。

4. 最後確認是否低於工程 tolerance 或 ISO tolerance。

5. Q&A 摘要

Q1. `zwst`、`zwtv` 分別代表什麼？

`zwst` 是 Zwicker stationary loudness，輸出單一穩態響度；`zwtv` 是 Zwicker time-varying loudness，輸出隨時間變化的響度曲線。

Q2. 兩者最大的差異在哪？

`zwst` 假設聲音穩定，重點是整段訊號的一個代表值；`zwtv` 保留時間軸，重點是瞬態與變化過程，因此多了時間濾波、非線性衰減與時間加權。

Q3. 測試中的誤差值代表什麼？

它代表本系統輸出和參考輸出的數值差距。參考可以是 Mosqito golden，也可以是 ISO Annex B 官方值；兩種比較回答的是不同問題。

Q4. 如何判斷誤差大不大？

先看絕對誤差是否有感知或顯示意義，再看相對誤差是否因 near-zero 參考值被放大，最後看是否低於工程 tolerance 或 ISO 的 +/-5% / 0.1 sone 合規容差。

Q5. Rust 系統是否比 Mosqito 精度更高？

不是。Rust 與 Python/numpy 都使用 f64；目前系統是對 Mosqito 做數值相容復刻。更高的是驗證密度、速度與決定性，不是單次浮點精度。

Q6. 誤差優化多少？

對 Mosqito golden 沒有要「優化」誤差，而是保持幾乎一致。真正可優化的是 `zwst` 對 ISO 官方基準的偏差：實驗顯示改用 ISO reference filter bank 後，Signal 3 可由 +0.82% 降到 +0.025%，Signal 5 可由 -0.76% 降到 0.000%。

Q7. 為什麼預設仍保留 Mosqito-compatible 路徑？

因為 golden 測試、逐階段除錯與使用者遷移一致性都依賴 Mosqito-compatible 行為。ISO reference mode 適合做成選配，而不是直接破壞預設相容性。

6. 結論

相容性

目前 Rust/AVX2 結果與 Mosqito golden 幾乎等價，端到端差異落在 1e-14 到 1e-9 的範圍。

合規性

對 ISO Annex B 的可見差異主要繼承自 Mosqito 的 stationary 前端選擇，仍在 ISO 容差內。

後續優化

若需要更貼近 ISO 官方 stationary 基準，可新增 `ZwstMode::IsoReference`，重用 `zwtv` 的 ISO 濾波器組。
